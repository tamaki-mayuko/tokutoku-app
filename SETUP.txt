徳徳ネット PWA版(バックエンド接続対応)
=========================================

■ 中身
- index.html           アプリ本体
- setup.sql            データベース設定(Supabaseに貼り付け)
- manifest.webmanifest ホーム画面追加用の設定
- sw.js                オフライン対応
- icon-*.png           アプリアイコン
- index-demo-backup.html 接続なしの体験版(バックアップ)

■ 2つのモード
- index.html の上部の SUPABASE_URL / SUPABASE_ANON_KEY が
  空のまま → 体験デモモード(データは保存されない)
  記入済み → 本接続モード(登録・依頼・徳・通知が全員で共有される)

■ 本接続のセットアップ(30分・無料)
1. https://supabase.com で無料アカウントを作り、
   「New project」でプロジェクトを1つ作る(リージョンは Tokyo)
2. 左メニューの「SQL Editor」を開き、setup.sql の中身を
   全部貼り付けて「Run」
3. 左メニューの「Project Settings → API」で
   ・Project URL
   ・anon public キー
   の2つをコピーし、index.html 上部の
   SUPABASE_URL / SUPABASE_ANON_KEY に貼り付けて保存
4. フォルダごと https://app.netlify.com/drop に
   ドラッグ&ドロップして公開

■ 動作確認(お試し合言葉)
- sakura … 高齢者(花子さん)。うめさんの依頼が見える
- ume   … 高齢者(うめさん)。依頼を出している側
- hina  … 家族(陽菜さん)。花子さんの見守り+仕送り徳
2台のスマホ(またはブラウザ2つ)で sakura と hina に入ると、
花子さんの行動が陽菜さん側に通知される流れを確認できます。

■ 会員の追加(導入会での運用)
Supabase の「Table Editor → members」で行を追加するか、
setup.sql 末尾のinsert文の書き方を使ってSQLで追加します。
合言葉(code)を紙に書いて本人に渡してください。

■ 大事な注意(PoC限定の割り切り)
- 今の設定は「合言葉を知っている人だけが参加する招待制」を
  前提にした簡易なアクセス設定です。URLと仕組みを外部に
  広く公開しないでください。
- 本格運用の前には、Supabase Auth による本人認証と
  行レベルセキュリティの厳格化が必須です。
- 個人情報は最小限(氏名・年齢のみ)にしてあります。
  住所や電話番号はPoC中はアプリに入れず、紙で管理してください。

■ 高齢者のスマホに入れる手順(導入会でスタッフが実施)
【iPhone / Safari】URLを開く → 共有ボタン → ホーム画面に追加
【Android / Chrome】URLを開く → メニュー(︙) → ホーム画面に追加
以後はホーム画面の赤い「徳」の印を押し、初回だけ合言葉を入力。
2回目からは合言葉不要で開きます(端末に記憶されます)。
